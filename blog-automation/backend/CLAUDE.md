# 세포아소프트 네이버 블로그 발행 하네스

이 폴더는 세포아소프트 네이버 블로그(blog.naver.com/sepoa0127) 콘텐츠를
일관된 규칙 위에서 생산하기 위한 하네스다. Claude Code로 이 폴더를 열면
아래 정책·파이프라인·가드레일을 항상 준수한다.

---

## 1. 컨텍스트 (항상 고정)

- **블로그**: blog.naver.com/sepoa0127
- **회사**: 세포아소프트 — B2B 지출관리(Spend Management) 솔루션 SingleSuite
- **제품군**: 전자구매/SRM, 경비지출관리, 출장관리, 전자계약, 전자인장
- **독자**: 구매·재무·총무 담당자, 의사결정권자 (B2B 실무자향)
- **톤**: 전문가향, 신뢰감, 과장 없는 사실 중심. 문제 → 해결 서사.

## 2. 파이프라인 (주제만 던지면 이 순서로 실행)

작업은 반드시 아래 순서를 따르며, [STOP] 지점에서는 사람 승인 없이
다음 단계로 넘어가지 않는다.

1. `/title-select`   → 제목 3안 제시  → **[STOP: 사람이 1개 선택]**
2. `/keyword-approve` → SEO 키워드 후보 → **[STOP: 사람이 승인]**
3. `/write-body`     → As-Is→To-Be 본문 초안을 drafts/ 에 저장
4. `/image-prompt`   → 썸네일·본문 이미지 프롬프트 → **[STOP: 이미지 텍스트 승인]**
5. `/evaluator`      → 검증 서브에이전트 통과 후에만 "발행 준비 완료" 선언

## 3. 산출물 규칙

- 본문 초안: `drafts/YYYY-MM-DD-{슬러그}.md`
- 이미지 프롬프트: `images/YYYY-MM-DD-{슬러그}-prompts.md`
- 각 글은 `spec.md`를 진실의 원천으로 삼는다 (없으면 먼저 작성 요청)

## 4. 가드레일 (반드시 지킴)

- **[STOP] 지점을 건너뛰지 않는다.** 앞 단계 승인 없이 다음으로 못 넘어간다.
- **금지**: 경쟁사 실명 언급, 검증 안 된 수치·효과, 과장 표현("업계 1위" 등).
- **제목**: 60자 이내, 질문형 권장.
- **본문 구조**: 반드시 As-Is(문제) → To-Be(세포아 솔루션이 바꾸는 것) 흐름.
- 모호하거나 위험 소지가 있으면 발행하지 말고 사람에게 물어본다.
- 검증(`/evaluator`)을 통과하지 못한 글은 "발행 준비 완료"로 표시하지 않는다.

## 5. 기존 Gemini Gems 자산 재활용

기존 Gems 지침(스톱포인트 3개, 60자 제목, As-Is→To-Be, 자가검증 체크리스트)은
이 파이프라인의 각 스킬 안에 이미 반영되어 있다. Gems를 별도로 쓸 경우에도
동일 규칙을 적용한다.
