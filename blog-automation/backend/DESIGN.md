# DESIGN.md — 세포아 블로그 이미지 시각 규칙

> 썸네일·본문 이미지의 **품질·신뢰 톤**을 매 글마다 일관되게 유지하기 위한 한 장짜리 규칙.
> image-prompt 스킬은 이 파일을 근거로 프롬프트를 만든다.
>
> **두 층으로 본다.** 톤·시인성·배색 절제·금지사항은 **항상 고정**이고,
> 레이아웃 구성·시각 모티프·강조색은 **글 주제에 맞춰 매번 새로** 고른다
> (매번 똑같은 그림이 나오지 않게 한다). 세부 선택 규칙은 image-prompt SKILL.md의
> "매번 새로 판단" 절을 따른다.

## 1. Visual Theme & Atmosphere
전문성과 신뢰. B2B 실무자가 "믿을 만하다"고 느끼는 정돈된 톤.
**밝고 깨끗한 배경**을 기본으로, 그 위에 **글 주제에서 나오는 모티프**를 은은하게
얹어 시그널을 준다. ("AI 기반 지출관리"가 실제로 글의 핵심일 때는 회로기판·AI
노드 모티프도 쓸 수 있지만, 모든 글의 기본값으로 반복하지 않는다 — §6 참조.)

## 2. Color Palette & Roles
- `--bg: #FFFFFF` 또는 `#F8FAFC`   /* 기본 배경, 흰색·아주 연한 회색 (항상 밝게) */
- `--primary: #0747A6`             /* SingleSuite 브랜드 프라이머리(파랑). 강조색의 기본값이자 대표색 */
- `--ink: #0F172A`                 /* 본문·제목 텍스트, 밝은 배경에서 잘 읽히는 다크 네이비 */
- **절제된 배색 원칙(고정)**: **한 이미지 안에서는 "밝은 배경 1색 + 강조 1색"** 으로
  절제한다. 한 그림에 강조색을 3색 이상 섞지 않는다(다크 네이비 텍스트·중립 회색은
  대비용 중립색이라 강조색 수에 넣지 않는다).
- **강조색의 색상은 글 주제에 맞춰 매번 고른다(가변)**. 파랑만 기계적으로 반복하지
  않는다. 대략의 기준(SKILL.md와 동일):
  - 신뢰·보안·안정 → 파랑 계열(`#0747A6`)
  - 효율·성과·활력 → 주황/초록 계열(주황 `#EA580C`, 초록 `#059669`)
  - 경고·리스크·통제 → 레드/앰버 계열(`#DC2626`)
  - 데이터·분석·정밀 → 청록/인디고 계열(`#0891B2`, `#4338CA`)
  → "밝은 배경 + 강조 1색"이라는 **절제 원칙은 고정**, 그 **강조 1색이 무엇인지는 글마다 달라진다.**
- 텍스트·라인에 다크 네이비를 인쇄용 CMYK로 변환할 때 자주색(퍼플)으로 틀어지지
  않도록 C100 M68~70 Y8~10 K45~58 기준을 지킨다.

## 3. Typography
- 국문: Pretendard, 대체 Noto Sans KR
- 영문·숫자: Roboto
- 이미지 내 텍스트는 굵고 짧게. 밝은 배경 위 다크 네이비(`--ink`)로 표기.
- 이미지 안 텍스트는 최소화하고 라벨은 짧게(2~4자) — 한글 오타·깨짐을 줄이기 위함.

## 4. Layout & Composition
- **레이아웃은 글 주제에 맞춰 매번 고른다.** 하나의 기본 구도를 모든 글에 반복하지
  않는다. image-prompt SKILL.md의 패턴 메뉴에서 고르거나 더 나은 구성을 새로 제안한다:
  - **흐름형**(입력→처리→결과), **중심 확산형**(중앙 개념에서 방사),
    **대비형**(어수선함↔정돈), **계층형**(위→아래 구조),
    **매핑형**(항목 간 연결/매칭), **타임라인형**(단계·시간 순서).
- **시각적 주인공 하나**가 화면을 지배하고 나머지는 그 주위로 정돈된다(공통 원칙).
  주인공이 무엇인지는 패턴에 따라 다르다(처리 노드·중심 개념·완성된 결과물 등).
- 제목은 **상단에 작게**, 도식이 나머지 공간을 크게 채운다. 비중은 서술로만
  표현하고 이미지 안에 "%"·숫자 같은 메타 텍스트를 넣지 않는다.
- **"As-Is"/"To-Be" 같은 글자 라벨이나 좌우를 "문제/해결"로 쪼개는 구도는 쓰지
  않는다.** 개선·대비가 필요하면 글자 라벨 없이 **시각적으로만** 대비시킨다
  (예: 어수선한 쪽 ↔ 정돈된 결과 쪽). 흐름·연결의 방향성은 화살표나 이어지는
  라인으로 명확히 표현한다.

## 5. Iconography
- Lucide 아이콘 계열, 라인(플랫) 스타일로 통일.
- 제품 모듈·요소는 노드(node) 형태로 표현 가능.

## 6. Imagery & Motif
- **시각 모티프는 글 주제에서 뽑는다.** 회로기판·AI칩을 기본값처럼 반복하지 않는다.
  예: 계약·인장 → 문서·도장·서명 / 원가·데이터 → 그래프·차트 / 조달·공급망 →
  연결망·물류 / 경비정산 → 영수증·전표 / 보안·통제 → 방패·자물쇠 / 통합·플랫폼 →
  허브·모듈 결합. **주제와 무관한 장식은 넣지 않는다.**
- **회로기판 모티프는 "AI가 글의 핵심"일 때만** 보조적으로 쓴다. 쓸 때도 밝은
  배경 위 가느다란 연한 파랑·회색 실선으로 은은하게(다크 배경의 발광 남발 금지).
- **강조는 주인공·핵심 연결·결과에 집중.** 시각적 주인공에만 강조색을 선명하게
  써 시선을 모으고, 나머지 요소는 중립 톤으로 차분하게 둔다. 주인공을 돋보이게
  하는 기법(발광·크기 대비·강조색 프레임·굵기 대비 등)은 구성에 맞게 고르고
  매번 같은 기법만 반복하지 않는다.
- 실사보다 정돈된 벡터·다이어그램 톤 우선.

## 7. Do's and Don'ts (가드레일)
- ✅ DO: 밝은 배경(흰색·연한 회색), **글 주제에 맞는 레이아웃·모티프·강조색을 매번
  선택**, 한 이미지 안에서는 "밝은 배경 + 강조 1색"의 절제된 배색, 시각적 주인공
  하나에 강조 집중, 짧은 텍스트, 정돈된 벡터 톤.
- ❌ DON'T: 다크 배경 기본값, 모든 글에 같은 구도·모티프·색(회로기판·좌우 흐름·
  파랑) 반복, 한 이미지에 강조색 3색 이상 혼용, "As-Is"/"To-Be" 글자 라벨,
  좌우를 문제/해결로 쪼개는 구도, 이미지 안 메타 텍스트(%, 숫자), 경쟁사 UI
  캡처, 과장 문구, 근거 없는 수치, 저작권 소지 이미지.

## 8. Accessibility
- 텍스트/배경 대비 충분히. 밝은 배경엔 다크 네이비(`--ink`) 텍스트로 대비를 확보한다.
- 요소 간 간격을 여유 있게 두고 산만하지 않게 — 시인성(가독성) 최우선.

## 9. 재사용 프롬프트 스니펫 (빈칸은 글마다 채운다)
```
세포아소프트 B2B 지출관리 블로그 썸네일.
흰색/아주 연한 회색(#F8FAFC) 밝은 배경 + {강조색 HEX} 강조 1색의 절제된 배색.
{레이아웃 패턴}: {한 줄 서술}.
  (예: 중심 확산형 — 중앙의 핵심 개념에서 관련 요소가 방사되는 구성)
시각 모티프: {주제에서 뽑은 소재}.  (회로기판·AI 노드는 AI가 글의 핵심일 때만)
{시각적 주인공}을 크고 인상적으로, 나머지는 그보다 작게 주인공 주위로 정돈.
흐름·연결은 굵기·스타일을 통일해 한 번에 읽히게.
"As-Is"/"To-Be" 같은 글자 라벨이나 좌우 대립 분할 구도는 넣지 않는다(대비는 시각적으로만).
Pretendard 굵은 국문 텍스트(다크 네이비 #0F172A): "{승인된 텍스트}".
정돈된 벡터 다이어그램 톤, 밝고 전문적·신뢰감, 시인성 우선(대비·간격 여유).
```
