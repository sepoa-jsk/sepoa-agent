# 이미지 프롬프트 — AI 계정과목 자동 추천 편

[썸네일]
- 이미지 내 텍스트: "OCR이 읽고, AI가 계정과목까지" / 보조 텍스트: "경비정산 자동화의 다음 단계"
- 프롬프트: 세포아소프트 B2B 지출관리 블로그 썸네일. 회로기판 배경, 다크 네이비(#0F172A)에 #0747A6 강조 2색 스킴. 좌측 As-Is(문제): 지로영수증 문서 아이콘과 물음표가 붙은 빈 계정과목 입력란(라인 스타일, 흐린 톤). 우측 To-Be(해결): 영수증에서 회로 라인이 뻗어나가 "계정과목" 노드에 연결되고 체크 표시가 켜진 구도(밝은 #0747A6 강조). Pretendard 굵은 국문 텍스트 중앙 상단: "OCR이 읽고, AI가 계정과목까지", 하단 보조 텍스트: "경비정산 자동화의 다음 단계". Lucide 계열 라인 아이콘, 정돈된 벡터 다이어그램 톤, 전문적·신뢰감. 3색 이상 혼용 금지, 다크 배경에 밝은 텍스트로 대비 확보.

[본문 이미지 1 — As-Is: 인식과 판단의 간극]
- 이미지 내 텍스트: "OCR은 읽기까지" / "계정과목 판단은 여전히 수작업"
- 프롬프트: 상(문제)/하(해결 예고) 분할 다이어그램, 다크 네이비(#0F172A) 배경에 #0747A6 강조 2색 스킴, 회로기판 모티프. 상단: 지로영수증(전기·수도·가스 고지서) 라인 아이콘에서 "금액·거래처 인식 완료" 체크까지는 회로 라인이 이어지지만, 그 다음 "계정과목?" 노드 앞에서 라인이 끊기고 사람 아이콘이 수작업으로 연결하는 모습. Pretendard 굵은 텍스트: "OCR은 읽기까지", 하단에 "계정과목 판단은 여전히 수작업". Lucide 16px 라인 아이콘, 정돈된 벡터 톤.

[본문 이미지 2 — To-Be: RAG 기반 계정과목 자동 추천]
- 이미지 내 텍스트: "과거 전표를 학습해 우리 회사 기준으로 추천" / 노드 라벨: "전북도시가스 → 수도광열비", "KT 인터넷 → 통신비", "한국전력 → 전력비"
- 프롬프트: 다크 네이비(#0F172A) 배경, #0747A6 강조 2색 스킴, 회로기판 배경. 중앙에 AI 노드(회로 문양의 칩 아이콘), 좌측에서 과거 전표 데이터 카드들이 회로 라인으로 AI 노드에 유입되고, 우측으로 거래처→계정과목 매핑 노드 3개가 연결: "전북도시가스 → 수도광열비", "KT 인터넷 → 통신비", "한국전력 → 전력비" (Pretendard, 짧은 라벨). 상단 굵은 텍스트: "과거 전표를 학습해 우리 회사 기준으로 추천". 노드 연결형 다이어그램, Lucide 라인 아이콘, 정돈된 벡터 톤.

[본문 이미지 3 — To-Be: 등록부터 결재까지 한 흐름]
- 이미지 내 텍스트: "촬영 → 인식 → 분류 → 계정 추천 → 적요 → 전표 → 상신" / 하단: "담당자는 확인·승인만"
- 프롬프트: 가로형 7단계 프로세스 플로우, 다크 네이비(#0F172A) 배경에 #0747A6 강조 2색 스킴, 회로기판 라인으로 단계 노드를 연결. 각 노드는 Lucide 계열 라인 아이콘(카메라, 스캔, 폴더 분류, 계정 태그, 텍스트 줄, 문서, 결재 도장)과 짧은 라벨: "촬영 → 인식 → 분류 → 계정 추천 → 적요 → 전표 → 상신". 마지막에 사람 아이콘과 체크 표시, 하단 텍스트: "담당자는 확인·승인만" (Pretendard 굵게). 정돈된 벡터 다이어그램 톤, 전문적·신뢰감.
