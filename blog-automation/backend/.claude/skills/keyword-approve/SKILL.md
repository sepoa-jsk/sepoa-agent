# 키워드 승인 (Step 2)

## 목적
확정된 제목 기준으로 (1) title-select 에서 미뤄둔 spec.md 를 마저 채우고,
(2) 네이버 검색 노출을 위한 키워드를 정리하고, (3) 그 키워드를 기반으로 한
본문 구조안(목차)을 제시한다.

## 먼저: spec.md 나머지 항목 채우기 (제목 단계에서 미뤄둔 것)
제목 단계(title-select)는 제목에 필요한 최소 정보(주제 한 줄·앵글·타깃 독자)만
파악했다. 이 단계에서 **확정된 제목과 원본(워드 초안 또는 주어진 주제)을 근거로**
spec.md 의 나머지 항목을 마저 채운다. 원본이 워드 초안이면 이때 본문을 제대로
읽어 근거를 확보한다.
- 확정 제목 (사람이 고른 제목을 그대로 기록)
- 핵심 메시지(세부)
- To-Be (세포아 솔루션이 바꾸는 것) / As-Is (독자가 겪는 문제)
- 타깃 독자 보강, 북극성(이 글이 노리는 것)
- 다룰 제품/모듈, 하지 말 것(이 글 한정 금지), 참고 링크·근거
- (핵심/보조 키워드와 배치안은 아래 키워드 작업 결과로 채운다)

검증되지 않은 수치·효과나 원본에 없는 사실은 지어내지 않는다.

## 규칙 (키워드)
- **핵심 키워드 1개** + **보조 키워드 3~5개**로 구성.
- 세포아 제품군 용어(전자구매, SRM, 경비지출, 출장관리, 전자계약, 전자인장,
  SingleSuite)를 우선 후보로 검토.
- 각 키워드가 본문 어디에 들어갈지(제목/첫 문단/소제목) 배치안을 함께 제시.
- 억지 키워드 삽입(키워드 스터핑) 금지.

## 규칙 (본문 구조안)
키워드 후보·배치안 다음에, 그 키워드를 기반으로 본문이 어떤 뼈대로 쓰일지
**목차 수준으로 간략히** 제시한다(각 항목 한 줄씩, 본문 내용을 미리 쓰지 않는다).
- **소제목은 3~4개.** 글 분량과 내용에 따라 판단한다.
- 소제목에 **확정 키워드(핵심/보조)가 자연스럽게 배치**되어야 하고, 어느
  소제목에 어떤 키워드가 들어가는지가 위 배치안과 일치해야 한다.
- 소제목은 **AEO 규칙에 따라 가능하면 질문형 또는 명확한 답이 있는 문구**로.
- **머릿말은 인사이트/문제 제기로 시작**(기능 나열 금지 — write-body 문체
  규칙과 일치), **맺음말은 핵심 메시지 한 줄**.

## 출력 형식
```
[키워드 후보]
- 핵심: ...
- 보조: ..., ..., ...
[배치안]
- 제목: ...
- 첫 문단: ...
- 소제목: ...
[본문 구조안]
머릿말: (한 줄 — 어떤 문제 제기/인사이트로 시작할지)
소제목 1: (소제목 문구)
소제목 2: (소제목 문구)
소제목 3: (소제목 문구)
(필요시 소제목 4)
맺음말: (한 줄 — 어떤 메시지로 마무리할지)
```

## spec.md 저장 (write-body 연계)
승인된 본문 구조안은 spec.md 에 **"본문 구조"** 항목으로 저장한다(머릿말/
소제목 1~4/맺음말을 그대로 기록). 다음 단계 /write-body 가 이 구조 그대로
본문을 쓴다. 사람이 구조를 수정하면 수정된 최종안을 저장한다.

## [STOP]
spec.md 나머지 항목을 채우고 키워드 후보·배치안·본문 구조안을 제시한 뒤
**반드시 멈춘다.** 사람에게 **"키워드와 본문 구조안을 확인하고 승인해주세요.
구조 수정을 원하시면 알려주세요."** 라고 안내한다. 사람이 구조(소제목 순서·
개수·문구)에 수정을 요청하면 반영해 다시 제시하고 STOP 한다. 승인 전에는
write-body로 진행하지 않는다.
