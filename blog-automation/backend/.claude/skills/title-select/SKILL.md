# 제목 선택 (Step 1)

## 목적
제목을 뽑는 데 꼭 필요한 최소 정보만 파악해 제목 3안을 만든다.
**이 단계는 가볍게 — spec.md 전체를 채우지 않는다.** 핵심 키워드·세부 메시지·
To-Be 등 나머지 spec 작성은 제목 확정 후 keyword-approve 단계로 미룬다.

## 이 단계에서 파악할 최소 정보 (이것만)
1. **주제 한 줄**: 이 글이 다루는 핵심을 한 문장으로.
2. **앵글**: 어떤 관점·문제의식으로 접근할지(제목의 후킹 포인트).
3. **타깃 독자**: 누구의 어떤 고민을 건드릴지.

- 워드 초안이 있으면 **전체를 정독하지 말고**, 위 세 가지를 판단할 수 있을
  만큼만 빠르게 훑는다(제목/도입부/소제목 위주). 본문 세부·수치·사례를
  꼼꼼히 정리하는 작업은 여기서 하지 않는다.
- 주제 한 줄만 주어진 경우도, 위 세 가지만 머릿속으로 잡고 바로 제목을 만든다.
- spec.md 에 위 세 항목(주제·앵글·타깃 독자)만 가볍게 적어도 되고, 굳이
  파일을 채우지 않고 제목 생성에 바로 써도 된다. **나머지 spec 항목은 비워 둔다.**

## 규칙 (제목 품질 — 절대 낮추지 않는다)
- 제목은 **60자 이내**.
- **질문형을 우선** 제안한다 (독자의 문제의식을 건드림).
- 3안은 서로 다른 앵글이어야 한다:
  1) 문제 제기형 (독자의 As-Is를 찌름)
  2) 해결 약속형 (To-Be를 암시)
  3) 숫자·구체형 (단, 검증된 수치만. 없으면 구체 상황으로)
- 과장·경쟁사 실명 금지.

## 출력 형식
```
[제목 3안]
1. (문제 제기형) ...
2. (해결 약속형) ...
3. (구체형) ...
```

## [STOP]
3안 제시 후 **반드시 멈춘다.** 사람이 번호를 고르기 전에는
다음 단계(keyword-approve)로 진행하지 않는다.
제목이 확정되면 keyword-approve 단계에서 spec.md 의 나머지 항목을 채운다.
