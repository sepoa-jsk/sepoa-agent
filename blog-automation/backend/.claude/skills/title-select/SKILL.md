# 제목 선택 (Step 1)

## 목적
spec.md의 주제·북극성·핵심 메시지를 근거로 제목 3안을 만든다.

## 규칙
- 제목은 **60자 이내**.
- **질문형을 우선** 제안한다 (독자의 문제의식을 건드림).
- 3안은 서로 다른 앵글이어야 한다:
  1) 문제 제기형 (독자의 As-Is를 찌름)
  2) 해결 약속형 (To-Be를 암시)
  3) 숫자·구체형 (단, 검증된 수치만. 없으면 구체 상황으로)
- 과장·경쟁사 실명 금지.

## 출력 형식
```
[제목 3안]
1. (문제 제기형) ...
2. (해결 약속형) ...
3. (구체형) ...
```

## [STOP]
3안 제시 후 **반드시 멈춘다.** 사람이 번호를 고르기 전에는
다음 단계(keyword-approve)로 진행하지 않는다.
