# 이미지 프롬프트 (Step 4)

## 목적
글을 대표하는 이미지(썸네일) **1개**의 "프롬프트 + 이미지 내 텍스트"만 산출한다.
본문 이미지는 만들지 않는다.
실제 이미지 생성은 사람이 Gemini나 ChatGPT에 프롬프트를 붙여넣어 직접 만들고
업로드하므로, 이 스킬의 산출물은 그대로 복사해 쓸 수 있어야 한다.
아래 "세포아소프트 대표이미지 표준 공식"과 `DESIGN.md`의 시각 규칙을 함께 따른다.

## 이미지 내 텍스트 산출 규칙 (임의로 짓지 않는다)
- **메인 텍스트 = spec.md의 "확정 제목"**. `spec.md`의 `## 확정 제목` 항목에서
  title-select 단계에서 사람이 고른 제목 문구를 그대로 가져와 이미지의 메인
  텍스트로 쓴다. 스킬이 새로 문구를 짓거나 요약·의역하지 않는다.
  - 제목이 길어 한 줄에 안 들어가면 **줄바꿈만** 한다(2줄 내외). 단어·조사를
    빼거나 표현을 바꾸지 않는다 — 문구 자체는 확정 제목과 완전히 동일해야 한다.
  - 줄바꿈은 반드시 **자연스러운 어절(단어) 단위**로 한다. 단어 중간에서
    끊는 어색한 분절은 금지한다.
  - `spec.md`에 확정 제목이 없으면 먼저 사람에게 확인을 요청하고 진행하지 않는다.
- **핵심 단어(강조 대상)**: `spec.md`의 `## 확정 키워드` 중 "핵심" 키워드와
  겹치는 단어를 제목 안에서 찾아 그 단어만 강조색으로 칠할 대상으로 표시한다.
  제목의 나머지 글자는 검정 기본색을 쓴다.
- **보조 텍스트/라벨(선택)**: 필요하면 `spec.md`의 "앵글" 또는 "핵심 메시지"
  중 가장 핵심적인 내용을 짧게 뽑아 쓴다. 원문 spec.md에 없는 내용을 새로
  지어내지 않는다.
- **효과 아이콘 라벨(3단 구성을 고른 경우에만)**: `spec.md`의 "핵심 메시지"·
  "To-Be" 항목에 있는 정성적 효과만 골라 쓴다. 검증되지 않은 구체 수치
  (예: "50% 절감", "3배 향상")는 절대 만들어내지 않는다.
- **이미지에 넣는 텍스트는 위 항목뿐**이다. 아래 "메타 텍스트 금지" 규칙에
  따라 비율·수치·여백 표시 같은 것은 텍스트로 넣지 않는다.

## 세포아소프트 대표이미지 표준 공식

### 1) 레이아웃 — 위아래 여백과 비중 (숫자를 이미지에 노출하지 않는다)
- 스킬 내부 설계 기준은 상단 여백 5% · 제목 영역 20% 이내 · 본문 도식 영역
  (나머지) 70% · 하단 여백 5%(합 100%)로 잡는다. **이 숫자는 프롬프트를
  작성할 때 비중을 가늠하기 위한 내부 기준일 뿐, 실제 Gemini/ChatGPT에
  보낼 프롬프트 문단에는 "%"나 숫자로 절대 쓰지 않는다** — 이미지 생성
  AI가 그 숫자를 글자로 그려 넣는 문제가 있었기 때문이다.
- 대신 프롬프트 문단에는 **서술형 문장**으로 바꿔 쓴다. 예: "제목은 이미지
  맨 위에 작게 배치하고 위아래로 충분한 여백을 둔다. 제목 아래로는 본문
  도식이 중앙부터 하단까지 대부분의 공간을 크게 차지하도록 그린다."
- 제목은 2줄 내외로 배치한다. 글자가 많으면 제목 영역이 상단의 작은
  비중을 넘지 않도록 폰트 크기를 줄이되(이때도 숫자로 표현하지 않는다),
  단어 중간에서 끊지 않고 자연스러운 어절 단위로만 줄바꿈한다.

### 2) 구성 — 2단 또는 3단 (글 내용에 따라 그때그때 판단)
- 프로세스와 도입 효과를 함께 보여줘야 하는 글 → **3단**: 제목 / 중앙 메인
  도식 / 하단 효과 아이콘 나열.
- 개념·구조 설명이 중심이라 효과 나열이 오히려 군더더기인 글 → **2단**:
  제목 / 중앙 메인 도식만.
- **메인 도식**: 핵심 흐름을 입력 → 처리 → 결과 순서로 좌→우로 도식화하되,
  **굵고 명확한 화살표로 끊김 없이** 이어지게 한다. 좌우를 대립시키는
  구도(예: "As-Is"/"To-Be" 라벨, 문제/해결 대비 분할)는 쓰지 않는다 —
  하나로 이어지는 파이프라인이어야 한다. 도입 효과는 흐름 끝 결과물이
  깔끔하게 완성된 모습으로 자연스럽게 드러낸다.
- **하단 효과 아이콘(3단일 때만)**: 도입 효과를 아이콘 + 짧은 라벨 4~5개로
  나열한다(예: 시간 단축, 업무 감소, 오류 최소화, 비용 절감, 생산성 향상 —
  실제 라벨은 이런 범주 중 `spec.md` 근거로 뒷받침되는 것만 정성적으로 골라
  쓴다).
- **상단 도식과 하단 효과의 시각적 연결**: 둘이 따로 노는 두 블록처럼
  보이지 않게, 다음 중 하나 이상의 방법으로 하나의 흐름처럼 묶는다 —
  (a) 완성된 결과물 노드에서 아래로 이어지는 연결선·화살표를 그려 하단
  효과 아이콘 열로 연결한다, (b) 하단 효과 아이콘들을 강조색 계열의 얇은
  테두리나 옅은 배경 프레임으로 묶어 하나의 세트처럼 보이게 한다.
- **정렬과 시선 흐름**: 모든 요소를 격자에 맞춰 정렬하고 아이콘 크기·
  간격을 통일해, 시선이 좌→우로 메인 흐름을 따라간 뒤 자연스럽게
  위→아래로 하단 효과까지 이어지게 한다.

### 3) 제목 강조색 — 글 내용으로 매번 판단
- 제목은 **검정 기본**색으로 쓰고, 핵심 단어(위 "핵심 단어" 규칙 참고)만
  강조색으로 칠한다.
- 강조색은 글의 주제·톤에 맞춰 아래 기준으로 고르고, 어떤 색을 왜 골랐는지
  프롬프트 문단에 명시한다:
  - 신뢰·보안·안정 주제 → 파랑 계열 `#0747A6`
  - 효율·활력·혁신·성과 주제 → 주황 계열 `#EA580C`
  - 원가·데이터·분석 주제 → 파랑 계열 `#0747A6`
- **강조색은 핵심 흐름에만 집중**한다 — 제목의 핵심 단어, 메인 도식의
  중앙 처리 노드와 화살표에만 강조색을 쓰고, 그 외 요소는 중립 톤으로
  눌러 시선이 강조색을 따라 흐름을 인식하도록 한다(하단 효과 아이콘을
  강조색으로 묶는 경우도 테두리·배경 정도로만 옅게 써서 메인 흐름의
  강조 노드보다 튀지 않게 한다).
- 배경의 은은한 회로 패턴 자체는 중립 톤으로 유지한다. 배경·본문은 밝은
  톤을 유지하고, 전체는 "밝은 배경 + 강조색 1가지"의 2색 스킴을 지킨다
  (검정·다크 네이비 텍스트는 대비용 중립색으로 별도 취급하며 3색 혼용으로
  세지 않는다).

### 4) 아이콘·톤 — 전문 솔루션 기업
- 아이콘은 **플랫 라인 스타일**로 통일한다(Lucide 계열, 16px 기준).
- 산만하지 않게 정돈된 벡터 다이어그램으로 마무리한다.
- 배경은 흰색/아주 연한 회색(`#FFFFFF`, `#F8FAFC`) 위에 회로기판 모티프를
  은은하게 얹는다(다크 배경 금지).
- 국문 폰트는 Pretendard(대체 Noto Sans KR).
- 세포아소프트는 B2B 전문 솔루션 기업이다 — 세련되고 신뢰감 있는 톤을
  유지하고, **시인성(가독성)을 최우선**으로 텍스트·배경 대비를 충분히
  확보하며 요소 간 간격을 여유 있게 둔다.

### 5) 공통 금지 사항
- 비율은 반드시 **1:1 정사각형**(예: 1024×1024). 다른 비율 금지.
- **이미지 안에 비율·수치·여백 표시("%", 숫자 라벨) 같은 메타 텍스트를
  그리지 않는다.** 이미지에 텍스트로 들어가는 것은 메인 제목·보조 텍스트·
  효과 아이콘 라벨뿐이다.
- "As-Is"/"To-Be" 같은 글자 라벨, 좌우 대립 구도 금지.
- 실사 사진, 경쟁사 UI 캡처, 과장 문구, 저작권 소지 이미지 금지.
- 검증되지 않은 구체 수치 금지.

## 프롬프트 작성 방식
- Gemini/ChatGPT 입력창에 **바로 붙여넣을 수 있는 자연어 한 문단**으로 쓴다.
- 항목 나열·마크다운 목록이 아니라 이어지는 문장으로 쓴다.
- 아래 내용을 빠짐없이 한 문단 안에 담는다:
  1. **1:1 정사각형 비율(1024×1024)**.
  2. **레이아웃은 서술형 문장으로** 명시한다 (예: "제목은 상단에 작게
     배치하고 위아래로 여백을 충분히 두며, 도식은 중앙~하단을 크게
     차지한다"). 제목은 2줄 내외·어절 단위 줄바꿈·글자 많으면 폰트 축소도
     서술형으로 포함한다. **"%"나 숫자로 여백·비중을 표현하지 않는다.**
  3. **메타 텍스트 금지 지시**를 문단에 그대로 포함한다 — "이미지 안에
     비율·수치·여백 표시(%, 숫자 라벨) 같은 메타 텍스트를 절대 그리지 말
     것. 오직 제목/부제/도식 라벨만 텍스트로 넣는다."
  4. **메인 텍스트(확정 제목 원문)** 를 큰따옴표로 그대로 인용하고, 핵심
     단어만 강조색으로 칠하고 나머지는 검정으로 쓴다는 지시를 포함한다.
     인용된 문구는 절대 의역·축약하지 않는다.
  5. 이번 글에 **2단/3단 중 무엇을 골랐는지와 그 이유**를 짧게 명시하고,
     선택한 구성대로 메인 도식(입력→처리→결과, 굵고 끊김 없는 화살표로
     연결된 좌→우 흐름)과, 3단이면 하단 효과 아이콘 4~5개(짧은 라벨 포함)
     를 함께 지시한다. **상단 도식과 하단 효과가 연결선·공통 프레임 등으로
     하나의 흐름처럼 보이게** 지시하고, 요소 정렬과 균일한 간격으로 시선이
     좌→우, 위→아래로 자연스럽게 흐르게 한다고 명시한다.
  6. **강조색과 선택 이유**를 명시한다(예: "신뢰·보안 주제이므로 파랑
     `#0747A6`을 강조색으로 사용"). 강조색은 핵심 흐름(중앙 처리 노드·
     화살표)에만 집중해서 쓰고 나머지 요소는 중립 톤으로 둔다고 명시한다.
  7. 플랫 라인 아이콘, 회로기판 모티프, Pretendard, 밝은 배경, 시인성 우선
     (대비·간격) 등 톤 지시.
  8. 금지 사항("As-Is"/"To-Be" 라벨, 좌우 대립 구도, 메타 텍스트, 실사·
     경쟁사 UI·과장 문구·저작권 소지 이미지, 근거 없는 수치).
- 보조 텍스트가 있으면 메인 텍스트 아래에 작게 배치한다고 명시한다.

## 출력 형식
```
[대표 이미지]
- 메인 텍스트(확정 제목): "..."
- 강조 단어 / 강조색: "..." (핵심 단어와 선택한 색·이유)
- 구성: 2단 | 3단 (선택 이유)
- 보조 텍스트: "..." (있는 경우만)
- 효과 아이콘 라벨: "...", "...", ... (3단일 때만)
- 프롬프트: (자연어 한 문단)
```
- 경로: `images/YYYY-MM-DD-{슬러그}-prompts.md` 에 저장.

## [STOP]
**이미지 내 텍스트**(메인 텍스트 + 강조 단어 + 보조 텍스트 + 효과 아이콘 라벨)를
먼저 사람에게 보여주고 승인받는다. 승인 전에는 최종 프롬프트를 확정하지
않으며 evaluator로 넘어가지 않는다.
