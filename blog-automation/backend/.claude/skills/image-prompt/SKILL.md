# 이미지 프롬프트 (Step 4)

## 목적
글을 대표하는 이미지(썸네일) **1개**의 "프롬프트 + 이미지 내 텍스트"만 산출한다.
본문 이미지는 만들지 않는다.
실제 이미지 생성은 사람이 Gemini나 ChatGPT에 프롬프트를 붙여넣어 직접 만들고
업로드하므로, 이 스킬의 산출물은 그대로 복사해 쓸 수 있어야 한다.
아래 "세포아소프트 대표이미지 표준 공식"과 `DESIGN.md`의 시각 규칙을 함께 따른다.

## 이미지 내 텍스트 산출 규칙 (임의로 짓지 않는다)
- **메인 텍스트 = spec.md의 "확정 제목"**. `spec.md`의 `## 확정 제목` 항목에서
  title-select 단계에서 사람이 고른 제목 문구를 그대로 가져와 이미지의 메인
  텍스트로 쓴다. 스킬이 새로 문구를 짓거나 요약·의역하지 않는다.
  - 제목이 길어 한 줄에 안 들어가면 **줄바꿈만** 한다(2줄 내외). 단어·조사를
    빼거나 표현을 바꾸지 않는다 — 문구 자체는 확정 제목과 완전히 동일해야 한다.
  - 줄바꿈은 반드시 **자연스러운 어절(단어) 단위**로 한다. 단어 중간에서
    끊는 어색한 분절은 금지한다.
  - `spec.md`에 확정 제목이 없으면 먼저 사람에게 확인을 요청하고 진행하지 않는다.
- **핵심 단어(강조 대상)**: `spec.md`의 `## 확정 키워드` 중 "핵심" 키워드와
  겹치는 단어를 제목 안에서 찾아 그 단어만 강조색으로 칠할 대상으로 표시한다.
  제목의 나머지 글자는 검정 기본색을 쓴다.
- **보조 텍스트/라벨(선택)**: 필요하면 `spec.md`의 "앵글" 또는 "핵심 메시지"
  중 가장 핵심적인 내용을 짧게 뽑아 쓴다. 원문 spec.md에 없는 내용을 새로
  지어내지 않는다.
- **효과 아이콘 라벨(3단 구성을 고른 경우에만)**: `spec.md`의 "핵심 메시지"·
  "To-Be" 항목에 있는 정성적 효과만 골라 쓴다. 검증되지 않은 구체 수치
  (예: "50% 절감", "3배 향상")는 절대 만들어내지 않는다.
- **이미지에 넣는 텍스트는 위 항목뿐**이다. 아래 "메타 텍스트 금지" 규칙에
  따라 비율·수치·여백 표시 같은 것은 텍스트로 넣지 않는다.

## 본문에서 "가장 극적인 변화 하나" 뽑기 (설명 아님 — 임팩트)
대표이미지는 정확한 설명 다이어그램이 아니라, **블로그 목록에서 스크롤을
멈추게 하는 한 방**이다. 임무는 "이 글의 모든 프로세스를 설명"이 아니라
"핵심 변화 하나를 크게 각인"이다. 프롬프트를 만들기 전에 **`drafts/` 의 실제
본문을 읽되**, 세부 단계를 전부 나열하지 말고 **가장 극적인 변화 하나**만
골라 크게 보여준다.

- **본문 프로세스를 전부 나열하지 않는다.** 단계가 많으면 **3개 덩어리로
  압축**한다.
  - 예(이번 글): 6단계(지로영수증 업로드→증빙 인식→계정과목 추천→적요 생성
    →전표 완성→결재)를 **"지로영수증 → AI → 완성된 전표"** 세 덩어리로 압축.
  - 세부 단계 설명은 **본문의 몫**이고, 대표이미지는 **핵심 변화 하나**만 크게
    보여준다.
- **"본문 내용 기반"은 유지**하되(아무 이미지가 아니라 이 글이 다루는 입력·
  처리·결과), 본문에서 **가장 극적인 변화를 골라 단순화**하는 방식으로 쓴다.
- 검증되지 않은 개념이나 본문에 없는 내용을 지어내지 않는다.

## 세포아소프트 대표이미지 표준 공식

### 1) 레이아웃 — 위아래 여백과 비중 (숫자를 이미지에 노출하지 않는다)
- 스킬 내부 설계 기준은 상단 여백 5% · 제목 영역 20% 이내 · 본문 도식 영역
  (나머지) 70% · 하단 여백 5%(합 100%)로 잡는다. **이 숫자는 프롬프트를
  작성할 때 비중을 가늠하기 위한 내부 기준일 뿐, 실제 Gemini/ChatGPT에
  보낼 프롬프트 문단에는 "%"나 숫자로 절대 쓰지 않는다** — 이미지 생성
  AI가 그 숫자를 글자로 그려 넣는 문제가 있었기 때문이다.
- 대신 프롬프트 문단에는 **서술형 문장**으로 바꿔 쓴다. 예: "제목은 이미지
  맨 위에 작게 배치하고 위아래로 충분한 여백을 둔다. 제목 아래로는 본문
  도식이 중앙부터 하단까지 대부분의 공간을 크게 차지하도록 그린다."
- 제목은 2줄 내외로 배치한다. 글자가 많으면 제목 영역이 상단의 작은
  비중을 넘지 않도록 폰트 크기를 줄이되(이때도 숫자로 표현하지 않는다),
  단어 중간에서 끊지 않고 자연스러운 어절 단위로만 줄바꿈한다.

### 2) 구성 — 2단 또는 3단 (글 내용에 따라 그때그때 판단)
- 프로세스와 도입 효과를 함께 보여줘야 하는 글 → **3단**: 제목 / 중앙 메인
  도식 / 하단 효과 아이콘 나열.
- 개념·구조 설명이 중심이라 효과 나열이 오히려 군더더기인 글 → **2단**:
  제목 / 중앙 메인 도식만.
- **메인 비주얼 = 시각적 주인공 하나**: 위에서 압축한 세 덩어리(입력 → AI →
  결과)를 배치하되, 단계를 촘촘히 늘어놓는 설명 다이어그램이 아니라 **중앙의
  AI 처리 노드를 시각적 주인공**으로 세운다. AI 노드는 **크고 인상적으로** —
  발광과 주변 회로 확산으로 이미지에서 **가장 눈에 띄는 요소**로 하되, **과하지
  않게 정돈된 확산**으로 그린다. **배경 회로는 은은하게 눌러** 주인공만 빛나게
  한다. 입력(예: 영수증)과 결과(예: 완성된 전표)는 **그보다 작게**, 주인공을
  향해 흐르도록 배치한다.
- **흐름 화살표는 굵고 일관되게**: 입력→AI→결과를 잇는 화살표는 **굵기·스타일을
  통일**해 흐름이 **한 번에 읽히게** 한다(방향·굵기가 제각각인 화살표 금지).
- **결과물에 "완성됐다"는 감각**: 결과물(완성된 전표 등)에 강조색 체크 표시·
  빛나는 테두리 등으로 "완성·해결됐다"는 감각을 부여해, 변화의 결말이 한눈에
  느껴지게 한다.
- **크게 쓰되 주인공 중심**: 제목 아래 공간을 시원하게 쓰되, 요소를 잘게 나눠
  여백을 메우려 하지 말고 **주인공(AI 노드)이 화면을 지배**하게 한다. 작고
  오밀조밀한 다단계 나열은 금지한다.
- 좌우를 대립시키는 구도("As-Is"/"To-Be" 라벨, 문제/해결 대비 분할)는 쓰지 않는다.
- **하단 효과 아이콘(3단일 때만, 보조 역할)**: 도입 효과를 **아이콘 + 짧은
  라벨(2~4자) 3개**로 나열한다(예: 시간 단축, 오류 감소, 업무 감소 — 이런
  범주 중 `spec.md` 근거로 뒷받침되는 것만 정성적으로 골라 쓴다). 하단 효과는
  **보조 역할**이므로 중앙 주인공(AI 노드)보다 **작고 차분하게**, 튀지 않게 둔다.
- **세 효과를 옅은 프레임(얇은 테두리·옅은 배경 밴드)으로 묶고**, 위쪽 **결과
  노드에서 그 프레임으로 이어지는 연결(연결선 또는 시각적 근접)을 명확히** 해,
  "결과 → 이런 효과"가 하나의 흐름으로 읽히게 한다.
- **상단 도식과 하단 효과의 시각적 연결**: 둘이 따로 노는 두 블록처럼
  보이지 않게, 다음 중 하나 이상의 방법으로 하나의 흐름처럼 묶는다 —
  (a) 완성된 결과물 노드에서 아래로 이어지는 연결선·화살표를 그려 하단
  효과 아이콘 열로 연결한다, (b) 하단 효과 아이콘들을 강조색 계열의 얇은
  테두리나 옅은 배경 프레임으로 묶어 하나의 세트처럼 보이게 한다.
- **시선 흐름**: 시선이 **먼저 중앙 주인공(AI 노드)에 꽂힌 뒤** 입력→결과의
  변화, 그리고 (3단이면) 아래 효과로 자연스럽게 이어지게 한다. 주인공을
  중심에 두고 나머지를 그 주위로 정돈한다.

### 3) 제목 강조색 — 글 내용으로 매번 판단
- 제목은 **검정 기본**색으로 쓰고, 핵심 단어(위 "핵심 단어" 규칙 참고)만
  강조색으로 칠한다.
- **강조는 핵심 단어 1~2개에만.** 여러 단어에 강조색을 흩어 칠하지 않는다
  (시선 분산 방지 — 강조가 많으면 아무것도 강조되지 않은 것과 같다).
- **제목 줄바꿈은 의미 단위로.** 1줄째와 2줄째가 각각 자연스러운 구(句)가
  되도록 프롬프트에 줄바꿈 위치를 명시한다(단어 중간·조사 뒤 어색한 분절 금지).
- 강조색은 글의 주제·톤에 맞춰 아래 기준으로 고르고, 어떤 색을 왜 골랐는지
  프롬프트 문단에 명시한다:
  - 신뢰·보안·안정 주제 → 파랑 계열 `#0747A6`
  - 효율·활력·혁신·성과 주제 → 주황 계열 `#EA580C`
  - 원가·데이터·분석 주제 → 파랑 계열 `#0747A6`
- **강조색은 핵심 흐름에만 집중**한다 — 제목의 핵심 단어, 메인 도식의
  중앙 처리 노드와 화살표에만 강조색을 쓰고, 그 외 요소는 중립 톤으로
  눌러 시선이 강조색을 따라 흐름을 인식하도록 한다(하단 효과 아이콘을
  강조색으로 묶는 경우도 테두리·배경 정도로만 옅게 써서 메인 흐름의
  강조 노드보다 튀지 않게 한다).
- 배경의 은은한 회로 패턴 자체는 중립 톤으로 유지한다. 배경·본문은 밝은
  톤을 유지하고, 전체는 "밝은 배경 + 강조색 1가지"의 2색 스킴을 지킨다
  (검정·다크 네이비 텍스트는 대비용 중립색으로 별도 취급하며 3색 혼용으로
  세지 않는다).

### 4) 아이콘·톤 — 전문 솔루션 기업
- 아이콘은 **플랫 라인 스타일**로 통일한다(Lucide 계열, 16px 기준).
- 산만하지 않게 정돈된 벡터 다이어그램으로 마무리한다.
- 배경은 흰색/아주 연한 회색(`#FFFFFF`, `#F8FAFC`) 위에 회로기판 모티프를
  은은하게 얹는다(다크 배경 금지).
- 국문 폰트는 Pretendard(대체 Noto Sans KR).
- 세포아소프트는 B2B 전문 솔루션 기업이다 — 세련되고 신뢰감 있는 톤을
  유지하고, **시인성(가독성)을 최우선**으로 텍스트·배경 대비를 충분히
  확보하며 요소 간 간격을 여유 있게 둔다.
- **정돈된 구성 (세련미)**: 프롬프트 전체에 다음을 명시한다 — *"여백을 충분히
  두고 요소 수를 최소화한 정돈된 구성. 장식 요소를 임의로 추가하지 말 것."*
  지시하지 않은 배지·리본·말풍선·불필요한 그래픽을 넣지 않는다.
- **한글 텍스트·라벨 최소화 (오타·깨짐 방지)**: 이미지 생성 AI가 긴 한글을
  자주 틀리므로("완염 씹" 같은 심각한 깨짐 발생), 이미지 안 한글은 **메인
  제목 + 입력/결과 라벨 + 효과 라벨** 정도로만 제한한다. **모든 단계·요소에
  라벨을 붙이지 않는다** — 입력과 결과 정도에만 라벨을 달고, 중앙 주인공과
  나머지는 아이콘·형태로 전달한다. 긴 문장·보조 설명 텍스트는 넣지 않는다.
- **노드 라벨은 2~4자**로 더 짧게 제한한다(글자가 적을수록 깨짐 확률이 준다).
  예: "지로영수증"(5자) 대신 **"영수증"**, 결과 라벨은 **"전표 완성"**.
- **프롬프트에 이미지에 들어갈 모든 한글 문구를 큰따옴표로 정확히 나열**하고,
  그대로 렌더링하라고 못박는다: *"아래 나열된 한글 문구를 철자 그대로, 한
  글자도 바꾸지 말고 정확히 렌더링하라. 나열되지 않은 텍스트는 어떤 것도
  추가하지 마라."* 각 라벨은 **정확한 철자로 크고 또렷하게** 그린다.
- 프롬프트 끝에 다음을 덧붙인다: *"한글이 어색하거나 존재하지 않는 단어로
  렌더링되지 않도록 특히 주의."*

### 5) 공통 금지 사항
- 비율은 반드시 **1:1 정사각형**(예: 1024×1024). 다른 비율 금지.
- **이미지 안에 비율·수치·여백 표시("%", 숫자 라벨) 같은 메타 텍스트를
  그리지 않는다.** 이미지에 텍스트로 들어가는 것은 메인 제목·보조 텍스트·
  효과 아이콘 라벨뿐이다.
- "As-Is"/"To-Be" 같은 글자 라벨, 좌우 대립 구도 금지.
- 실사 사진, 경쟁사 UI 캡처, 과장 문구, 저작권 소지 이미지 금지.
- 검증되지 않은 구체 수치 금지.

## 프롬프트 작성 방식
- Gemini/ChatGPT 입력창에 **바로 붙여넣을 수 있는 자연어 한 문단**으로 쓴다.
- 항목 나열·마크다운 목록이 아니라 이어지는 문장으로 쓴다.
- 아래 내용을 빠짐없이 한 문단 안에 담는다:
  1. **1:1 정사각형 비율(1024×1024)**.
  2. **레이아웃은 서술형 문장으로** 명시한다 (예: "제목은 상단에 작게
     배치하고 위아래로 여백을 충분히 두며, 도식은 중앙~하단을 크게
     차지한다"). 제목은 2줄 내외·어절 단위 줄바꿈·글자 많으면 폰트 축소도
     서술형으로 포함한다. **"%"나 숫자로 여백·비중을 표현하지 않는다.**
  3. **메타 텍스트 금지 지시**를 문단에 그대로 포함한다 — "이미지 안에
     비율·수치·여백 표시(%, 숫자 라벨) 같은 메타 텍스트를 절대 그리지 말
     것. 오직 제목/부제/도식 라벨만 텍스트로 넣는다."
  4. **메인 텍스트(확정 제목 원문)** 를 큰따옴표로 그대로 인용하고, **핵심
     단어 1~2개에만** 강조색을 칠하고 나머지는 검정으로 쓴다는 지시를 포함한다
     (여러 단어에 흩어 칠하지 않는다). **줄바꿈 위치를 의미 단위로 명시**해
     1줄째·2줄째가 각각 자연스러운 구가 되게 한다. 인용된 문구는 절대 의역·
     축약하지 않는다.
  5. **핵심 변화 하나를 압축**해 지시한다. 본문 프로세스를 다 나열하지 말고
     3개 덩어리(예: "영수증 → AI → 전표 완성")로 압축하고, **중앙 AI 노드를
     시각적 주인공**으로 세운다고 명시한다 — 발광·주변 회로 확산으로 가장 눈에
     띄게 하되 **과하지 않게 정돈된 확산**, 배경 회로는 은은하게 눌러 주인공만
     빛나게. 입력·결과는 그보다 작게 주인공을 향해 흐르게 한다.
  6. **흐름 화살표를 굵고 일관된 스타일**로, 입력→AI→결과가 **한 번에 읽히게**
     지시한다. **결과물에 "완성됐다" 감각**(강조색 체크·빛나는 테두리)을 부여
     하고, 작고 오밀조밀한 다단계 나열은 금지한다고 명시한다.
  7. 3단이면 **하단 효과 아이콘 3개(라벨 2~4자)**를 보조 역할로 지시한다 —
     옅은 프레임으로 묶고, **위쪽 결과 노드에서 그 프레임으로 이어지는 연결
     (연결선·시각적 근접)을 명확히** 해 "결과→효과"가 하나로 읽히게. 중앙
     주인공보다 작고 차분하게 둔다.
  8. **한글 오타·깨짐 억제**를 강하게 지시한다 — 이미지에 들어갈 **모든 한글
     문구를 큰따옴표로 정확히 나열**하고 *"아래 나열된 한글 문구를 철자 그대로,
     한 글자도 바꾸지 말고 정확히 렌더링하라. 나열되지 않은 텍스트는 어떤 것도
     추가하지 마라."* 를 넣는다. 노드 라벨은 **2~4자**로 제한. 문단 끝에 *"한글이
     어색하거나 존재하지 않는 단어로 렌더링되지 않도록 특히 주의."* 를 덧붙인다.
  9. **강조색과 선택 이유**를 명시한다(예: "신뢰·보안 주제이므로 파랑
     `#0747A6`을 강조색으로 사용"). 강조색은 핵심 흐름(중앙 처리 노드·
     화살표)과 결과 노드에 집중해서 쓰고 나머지 요소는 중립 톤으로 둔다고
     명시한다.
  10. 플랫 라인 아이콘, 회로기판 모티프, Pretendard, 밝은 배경, 시인성 우선
     (대비·간격) 등 톤 지시와 함께, **"여백을 충분히 두고 요소 수를 최소화한
     정돈된 구성. 장식 요소를 임의로 추가하지 말 것."** 을 명시한다.
  11. 금지 사항("As-Is"/"To-Be" 라벨, 좌우 대립 구도, 메타 텍스트, 실사·
     경쟁사 UI·과장 문구·저작권 소지 이미지, 근거 없는 수치).
- 보조 텍스트가 있으면 메인 텍스트 아래에 아주 짧게 배치한다고 명시한다.

## 출력 형식
```
[대표 이미지]
- 메인 텍스트(확정 제목): "..."
- 강조 단어 / 강조색: "..." (핵심 단어와 선택한 색·이유)
- 구성: 2단 | 3단 (선택 이유)
- 보조 텍스트: "..." (있는 경우만)
- 효과 아이콘 라벨: "...", "...", ... (3단일 때만)
- 프롬프트: (자연어 한 문단)
```
- 경로: `image-prompts/YYYY-MM-DD-{슬러그}-prompts.md` 에 저장.
  (실제 이미지 파일이 아니라 프롬프트 텍스트이므로 `images/` 가 아닌
  `image-prompts/` 폴더에 저장한다.)

## 저장 시점 (STOP 전에 즉시 저장)
프롬프트를 작성하면 **[STOP]에서 승인받기 전에 즉시** 위 경로
(`image-prompts/YYYY-MM-DD-{슬러그}-prompts.md`)에 **파일로 저장한다.**
- 승인 여부와 무관하게 파일을 먼저 만든다 — 화면의 "프롬프트 보기/복사"
  버튼이 승인 전에도 파일을 읽을 수 있어야 하기 때문이다.
- 사람이 수정을 요청하면 프롬프트를 고쳐 **같은 파일을 덮어쓴** 뒤 다시 STOP.

## [STOP]
프롬프트 파일을 저장한 뒤, **이미지 내 텍스트**(메인 텍스트 + 강조 단어 +
보조 텍스트 + 효과 아이콘 라벨)를 사람에게 보여주고 이렇게 안내한다:
**"이미지 내 텍스트를 확인해 주세요. 수정 요청이 있으면 반영해 파일을
덮어쓰고 다시 제시합니다. 승인하시면 다음 단계로 진행합니다."**
- 수정 요청 시: 프롬프트를 고치고 같은 파일을 덮어쓴 뒤 다시 STOP 한다.
- 승인 전에는 다음 단계(evaluator)로 넘어가지 않는다.
